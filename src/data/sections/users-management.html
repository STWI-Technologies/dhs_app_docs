Users Management

Overview

The User Management section provides a clear overview of everyone who has access to your DHS account. From here, owners can review user details such as contact information, assigned roles, and crew memberships, as well as make updates when necessary. It’s the central place to keep user information current and ensure that each team member has the right level of access.

From this section you can:

  • Add new users with a role, contact information, and a password.
  • Edit existing users to update their profile, change their role, or reassign them.
  • Disable users who should no longer have access to the platform (without deleting their data).
  • Enable previously disabled users to restore their access.
  • Send password reset emails so users can set a new password.
  • Search and filter users by name, email, or role.
  • Manage profile pictures by uploading or removing avatar images.

Important: Disabling a user prevents them from logging in but preserves all their historical data (past jobs, appointments, timesheets, etc.). You can re-enable a disabled user at any time to restore their access.

Users List View

The Users List View is the main view for managing all your users. To access it, navigate to Menu > Users.

Tabs

At the top of the list, a segmented control lets you switch between two views:

Tab

What It Shows

  • A New User button — click to open the add user form.

User Roles

Every user in the system is assigned a role that determines what they can see and do. The four roles are:

Role

Tag Color

Description

Owner

Gray

The account owner. This role is automatically assigned to the person who created the account. There is typically one owner per account.

Admin

Green

Full administrative access to the platform. Admins can manage users, crews, settings, and all business operations. This is the highest role that can be assigned to other users.

Support

Yellow

Support staff role. Intended for office and dispatch personnel who manage scheduling, client communication, and day-to-day operations but may not need full admin access.

Crew Member

Blue

Standard field worker role. Intended for technicians, installers, and other crew members who primarily work on jobs in the field. This is the default role when creating a user via Quick Add.

1. Adding a New User

To add a new user to your account:

Step 1: Click the Add User button at the top of the Users List View. A panel will slide open from the right side with the title Add New User.

Step 2: Fill in the user's profile information.

Password Requirements

The password must meet all of the following criteria:

  • Minimum 10 characters long
  • Maximum 128 characters long
  • At least one uppercase letter (A-Z)
  • At least one lowercase letter (a-z)
  • At least one number (0-9)
  • At least one special character (!@#$%^&*()_+-=[]{}|;:'",./<>?/~`)

If the password does not meet these requirements, you will see the error: "Password must contain at least one uppercase letter, one lowercase letter, one number, and one special character."

Tip: You can use the Generate button (star icon next to the password field) to automatically create a strong password that meets all requirements. Use the Copy button (clipboard icon) to copy the generated password so you can share it with the user. You will see a confirmation: "Password copied."

A note below the password field reads: "The password will be sent to the recipient email."

Step 3: Click Add User at the bottom of the panel to create the user account.

On success, you will see the toast message: "User successfully created." The panel closes and the new user appears in the list.

If creation fails, you will see: "Failed to create user."

2. Editing a User

To edit an existing user, click the Edit (pencil icon) button on the user's row, or click the user's name in the table. A panel will slide open from the right side with the title Edit User.

The edit form shows the same fields as the create form (Profile Picture, First Name, Last Name, Email, Phone, Role, Timezone, Description), pre-filled with the user's current information. The Set Password and Confirm Password fields are not shown in edit mode — instead, a password reset section appears.

Password Reset

Below the main form fields, a section titled Reset Password appears with the description: "Send a password reset link to the user's email address."

Note: The password reset section is not shown when editing your own profile — it only appears when editing another user's profile.

2.1. Profile Picture Management

Each user can have a profile picture (avatar) that appears throughout the platform — in the users list, crew member lists, appointment assignments, and more.

2.2. Uploading a Profile Picture

  1. Open the user's profile in add or edit mode.
  2. Click Upload Image next to the avatar placeholder.
  3. Select an image file from your computer.
  4. The image is displayed as a preview in the form.

2.3. Removing a Profile Picture

The remove button appears next to the image. Click it to clear the avatar. The user's display will revert to showing the default avatar.

3. Disabling a User

Disabling a user prevents them from logging into the platform while preserving all their historical data. This is the recommended approach when someone leaves your team or should temporarily lose access.

There are two ways to disable a user:

3.1. From the Users List View

  • Make sure you are on the Active Users tab.
  • Click the Disable (user with X icon) button on the user's row.
  • A confirmation dialog appears with the title Disable User and the message: "Are you sure you want to disable this user? This will prevent them from logging in and accessing the app."
  • Click Disable User (red button) to confirm, or Cancel to go back.

3.2. From the Edit Panel

  1. Open the user's profile in edit mode.
  2. Click the Disable User button in the top-right corner of the panel.
  3. A black confirmation popover appears with the same message: "Are you sure you want to disable this user? This will prevent them from logging in and accessing the app."
  4. Click Disable User to confirm, or Cancel to dismiss.

The user moves from the Active Users tab to the Disabled Users tab.

Warning: You cannot disable a user who has active job assignments. If a user is currently assigned to an active job, the disable button will be grayed out and disabled. The tooltip reads: "Cannot Disable User With Active Job Assignments."

4. Enabling a Disabled User

To restore access for a disabled user:

4.1. From the Users List View

  1. Switch to the Disabled Users tab.
  2. Click the Enable (undo icon) button on the user's row.
  3. A confirmation dialog appears with the title Enable User and the message: "Do you want to enable this user? They will restore app access."
  4. Click Enable User to confirm, or Cancel to go back.

4.2. From the Edit Panel

  1. Open the disabled user's profile in edit mode.
  2. Click the Enable User button in the top-right corner.
  3. A black confirmation popover appears with the message: "Are you sure you want to enable this user?"
  4. Click Enable User to confirm, or Cancel to dismiss. The button shows "Enabling User" while processing.

On success: "User successfully enabled." The user moves from the Disabled Users tab back to the Active Users tab and can immediately log in again.

Quick Add User

You can create a user quickly from other sections of the platform (such as when adding members to a crew) using the Quick Add feature. The Quick Add modal is titled Quick Add New User and shows a simplified form with an informational note: "Quickly add a new user without leaving this page. You can complete all the details later."

Quick Add Form Fields

Field

Description

First Name

The user's first name.

Last Name

The user's last name.

Email

Must be unique. The system checks for duplicates.

Phone

Phone number with country selector (defaults to US).

Role

Defaults to Crew Member. When creating from a Crew context, only Admin and Crew Member are available.

The user's timezone.

The Quick Add form does not include the description field, profile picture, or password — a password is automatically generated, and the user can set their own via email. The locale defaults to English.

On success, you will see the toast message: "User created using Quick Add" with a View User button. Clicking View User navigates to the Users List View with the new user's edit panel already open, so you can add more details if needed.

Searching and Filtering

Search

The search bar at the top of the Users List View lets you filter users by keyword. The search applies to user names and email addresses. As you type, the list updates automatically.

Filter by Role

The Role filter is a multi-select dropdown with four options:

  • Owner
  • Admin
  • Support
  • Crew Member

Select one or more roles to see only users with those roles. This is useful for quickly finding all your admins or all your field crew members.


When filters or search terms change, the pagination resets to page 1.

Cross-Platform Usage

Users appear throughout the entire DHS platform:

Crews

Users can be assigned as members of one or more crews. Within a crew, one user is designated as the Crew Lead. When viewing the Users List View, the Crew column shows which crews each user belongs to. Disabled users still appear in crew member lists but are marked as "(Disabled)" and cannot be set as crew lead.

Appointments

When scheduling appointments, crews (and their member users) are assigned to handle the work. The system uses crew availability settings to determine when team members are available for scheduling.

Jobs

A user's crew determines their job assignments. The system uses a flag to stop you from disabling users who are still working

Timesheets

Each user's work hours are tracked through timesheets. The timesheet system records when users clock in and out of jobs.

Reports and Dashboard

User data feeds into various reports including performance metrics, feedback ratings, and activity summaries. Users can be filtered in reports to see individual or team performance.

Chat and Messages

Users can communicate through the platform's messaging system. Each user's name and avatar appear in conversations and notification lists.
