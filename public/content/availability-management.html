Availability Management

Overview

The Availability section is where you configure when your business is open for service and when it is not. Think of it as your business calendar — it tells the platform which days and hours your team is available to work, and which dates, months, or seasons should be blocked off entirely.

Availability affects how appointments are scheduled across the platform. When someone creates an appointment, the system uses these settings to determine which time slots are available. There are two levels of availability:

  • Global Availability — Sets the overall work schedule for your entire business. This is the default schedule that applies to all crews unless overridden.
  • Per Crew Availability — Allows you to set a custom schedule for individual crews. A crew's availability must fall within the global hours (a crew cannot be scheduled outside the business's overall hours).

From this section you can:

  • Enable or disable work days — Toggle individual days of the week on or off.

  • Set time slots — Define one or more working time ranges per day (for example, morning and afternoon shifts).

  • Add exceptions — Block off specific date ranges, entire months, or full seasons when your business is unavailable.

  • Configure per-crew schedules — Give individual crews their own availability within the global hours.

Important: Availability is not a standalone list of records like Services or Products. It is a configuration page — you set it up once and adjust it as your business needs change. There are no items to archive or delete. Changes are saved automatically as you make them.

Accessing Availability

There are two ways to access Availability settings:

  1. Main Availability Page — Navigate to Menu > Availability. This is the full-page view with the title "Availability" at the top, the schedule type toggle, and all configuration sections.

  1. Settings > Availability — Navigate to Menu > Settings > Availability. This shows the exact same form embedded within the Settings page. Any changes you make in either location are reflected in both.

Tip: If you see an Availability summary on a crew's detail view, you can click the View Availability Settings link to jump directly to the Availability page with that crew pre-selected in Per Crew mode.

Schedule Types: Global vs. Per Crew

At the top of the Availability page, you will see a segmented control (toggle) with two options:

Option

What It Does

Global

Sets the overall service availability for your entire business. This is the default schedule. The description reads: "Set your overall service availability." Below the toggle, the section is titled General Schedule.

Per Crew

Lets you set a custom schedule for a specific crew. The description reads: "Set crew service availability." After selecting this option, a Select Crew dropdown appears where you choose which crew to configure.

Note: You cannot switch between Global and Per Crew while any changes are being saved. The toggle is disabled during save operations to prevent data conflicts.

Selecting a Crew (Per Crew Mode)

When you switch to Per Crew, a dropdown labeled Select Crew appears. This dropdown lists all the crews in your account (up to 100 crews are loaded). Select the crew you want to configure. Once selected, the page loads that crew's existing availability settings so you can review and modify them.

Important: In Per Crew mode, a crew's time slots must fall within the global hours. If you try to set a crew's time slot outside the global availability window, you will see a warning message: "Crew slot is outside global hours." The system will show you what the global hours are for that day so you can adjust accordingly.

Enabling and Disabling Days

Each day has a toggle switch next to it. Here's how it works:

  • Turning a day ON — Flipping the toggle to the "on" position enables that day and automatically creates an empty time slot for you to fill in.

  • Turning a day OFF — Flipping the toggle to the "off" position removes all time slots for that day and immediately saves your change. You will see a success message: "Availability updated successfully."

Warning: Turning off a day is immediate and removes all time slots for that day. If a day had multiple time slots (such as a morning and afternoon shift), all of them will be removed at once.

Setting Time Slots

For each enabled day, you can define one or more time slots. Each time slot represents a continuous block of working hours.

Each time slot row has:

Element

Description

From time selector

The start time of the work period. Click the field to open a time picker modal where you select hours and minutes, then AM or PM.

To time selector

The end time of the work period. Same time picker interface as the From field.

Apply button (checkmark icon)

Saves this time slot. Only enabled when both From and To times are set and valid. Disabled if any other time slot on any day has incomplete times.

Discard button (X icon)

Reverts the time slot to its previously saved state. If this is a brand-new slot that has never been saved, discarding it removes it entirely.

Delete button (trash icon)

Removes a previously saved time slot. Only shown when a day has more than one time slot. Saves immediately.

Adding Multiple Time Slots Per Day

To add a second (or third, etc.) time slot on the same day, click the + Add Time Slot button that appears below the existing slots for that day.

Note: You can only add one new time slot at a time across all days. While you have an unsaved (pending) time slot, the + Add Time Slot buttons on all other days are disabled. You must either Apply or Discard the pending slot before you can add another one.

Exceptions Section

The Exceptions section lets you define periods when your business (or a specific crew) is completely unavailable, regardless of the regular work days schedule. This is useful for holidays, vacation periods, seasonal shutdowns, or any other time you need to block off the calendar.

The Exceptions area appears below the Work Days section with the heading Exceptions and an Add button to the right.

There are three types of exceptions:

Exception Type

Tab Label

What It Does

By Day Range

Blocks a specific range of dates (for example, December 24 through January 2). Can optionally repeat every year.

Blocks one or more entire months (for example, all of January and February). Only one set of month exceptions is allowed at a time.

Blocks one or more seasons: Winter, Spring, Summer, or Fall. Only one set of season exceptions is allowed at a time.

Adding an Exception

Step 1: Click the Add button in the Exceptions header. A form will appear with tabs at the top for the available exception types.

Step 2: Select the type of exception you want to create by clicking one of the tabs: By Day Range, By Month, or By Season.

Note: If you already have a month exception configured, the By Month tab will not appear (you can only have one set of month exceptions). The same applies to season exceptions — once configured, the By Season tab disappears. Date range exceptions have no limit; you can add as many as you need. When both month and season exceptions are already configured, the message "Month and season exceptions configured" appears.

Editing a Date Range Exception

After a date range exception is created, it appears as a card showing the date range and whether it recurs annually. Each card has:

  • An Edit button (pencil icon) — Click to open the exception in edit mode, where you can change the From date, To date, and the Annual checkbox.

  • A Delete button (trash icon) — Click to delete the exception (a confirmation dialog appears first).

When editing:

  • Modify the fields as needed.
  • Click Save to apply changes (the button shows "Saving..." while processing).
  • Click Cancel to discard your changes and return to the view mode.

Month Exceptions (By Month)

Month exceptions let you block one or more entire calendar months.

When you select the By Month tab, you see:

  • 12 circular month buttons arranged in a row — Jan, Feb, Mar, Apr, May, Jun, Jul, Aug, Sep, Oct, Nov, Dec. Each button is a small circle showing the three-letter month abbreviation.

  • Click a month to select it (the button turns blue with white text). Click again to deselect (returns to gray).
  • A description below updates in real time: "All services will be unavailable: [selected month names]."

For a new month exception (created from the Add form):

  • Select the months you want to block.
  • Click Apply Changes to save (the button shows "Applying..." while processing).
  • Click Cancel to discard and close the form.

For an existing month exception (already saved):

  • The exception appears as a card titled Exception with a number (for example, "Exception #1").
  • You can toggle months on or off. When changes are detected, Apply Changes and Cancel buttons appear.
  • A Delete button (trash icon) in the card header lets you remove all month exceptions.

Cross-Platform Usage

Availability is not just a settings page — it directly affects how other parts of the platform work.

Appointments and Scheduler

When creating or editing an appointment, the system checks the crew's availability to determine which time slots can be offered. The appointment form uses the availability data to:

  • Show available time slots based on the assigned crew's schedule.
  • Check for date exceptions (date ranges, months, seasons) and block those dates.
  • Display an Availability Info note so users understand why certain times may not be available.

If a user needs to schedule an appointment outside the normal availability window, the system supports an Override Availability option. When this is enabled, the appointment can be created even if it falls outside the configured hours or during an exception period.

Tip: The Override Availability option is useful for emergency calls, special client requests, or one-off situations where the regular schedule doesn't apply.

Crews

Each crew has an Availability summary displayed on its detail view. This summary shows:

  • General Availability — A list of enabled days with their time ranges, each marked with a green dot. For example: "Monday, 9:00 AM - 5:00 PM."
  • Exceptions and Special Conditions — A list of all active exceptions, each marked with an orange dot. This includes date range exceptions (for example, "Not Available Dec 24 - Jan 2"), month exceptions (for example, "Not Available during January"), and season exceptions (for example, "Not Available during Winter").

If no availability data has been configured for a crew, the summary shows: "No availability data."

Clicking View Availability Settings on a crew's summary navigates you to the Availability page with that crew automatically pre-selected in Per Crew mode.

Jobs and Estimates

When scheduling work through jobs, the system uses crew availability data to calculate available time slots for the job's duration. Exception dates, monthly exceptions, and seasonal exceptions are all factored into the calculation.

Step-by-Step: Setting Up Availability from Scratch

If this is your first time configuring availability, follow these steps:

Step 1: Set Global Work Days

  1. Navigate to Menu > Availability (or Settings > Availability).
  2. Make sure Global is selected in the toggle at the top.
  3. For each day your business operates, flip the toggle switch to the "on" position.
  4. Set the From and To times for each enabled day. Click the time fields to open the time picker.
  5. Click the Apply (checkmark) button next to each time slot to save it.
  6. If a day has split shifts (for example, 8:00 AM - 12:00 PM and 1:00 PM - 5:00 PM), click + Add Time Slot and repeat steps 4-5 for the additional slot.

Step 2: Add Any Exceptions

  1. Scroll down to the Exceptions section.
  2. Click Add.
  3. Choose the exception type: By Day Range, By Month, or By Season.
  4. Fill in the details (dates, months, or seasons to block).
  5. Click Create Exception or Apply Changes to save.
  6. Repeat for each exception you need. Remember you can have unlimited date range exceptions but only one set of month exceptions and one set of season exceptions.

Step 3: Configure Per-Crew Schedules (Optional)

  1. Switch to Per Crew using the toggle at the top.
  2. Select a crew from the Select Crew dropdown.
  3. Set the crew's work days and time slots, following the same process as for Global availability.
  4. Add any crew-specific exceptions.
  5. Repeat for each crew that needs a custom schedule.

Important: Remember that a crew's time slots cannot extend beyond the global hours. If the global schedule says Monday is 9:00 AM - 5:00 PM, a crew cannot be set to 8:00 AM - 6:00 PM on Monday. You'll see the warning "Crew slot is outside global hours" if you try.
